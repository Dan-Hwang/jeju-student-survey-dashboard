# 발표형 데이터 스토리 상호작용 보완 설계

## 문제 정의

현재 발표형 스토리는 막대 호버 시 위치와 투명도만 바뀌며, 사용자가 클릭해도 선택 상태나 데이터가 변하지 않는다. 이를 인터랙티브 화면이라고 부르지 않고, 발표자가 핵심 문제를 선택하면 관련 근거와 시냅스팟 기능이 실제로 교체되는 탐색 화면으로 보완한다.

기존 Google Sheets 자동 집계, 상세 통계, 자유 의견, PDF 다운로드는 유지한다. 이번 변경은 발표 흐름 상단에만 적용하며 설문 파서와 원본 데이터 구조는 바꾸지 않는다.

## 성공 기준

- 상단에 `전체 응답`, `이동·동행`, `정보 탐색` 선택 컨트롤이 보인다.
- 선택 후 제목, 설명, 근거 그래프, 핵심 해석, 제품 미리보기가 같은 주제에 맞게 함께 바뀐다.
- 선택 상태는 Streamlit 세션 상태에 유지된다.
- 마우스를 빼도 선택 결과가 사라지지 않는다.
- 호버 효과를 제거해도 모든 기능과 정보가 그대로 동작한다.
- 390px 모바일에서는 선택 컨트롤과 콘텐츠가 한 열로 표시되고 가로 스크롤이 생기지 않는다.
- 자동 테스트가 선택값별 서로 다른 데이터와 제품 연결을 검증한다.

## 사용자 흐름

### 전체 응답

- 전체 응답 수와 한국인/외국인 구성을 보여준다.
- 두 핵심 문제를 나란히 요약한다.
- 발표의 시작점이며 기본 선택값이다.

### 이동·동행

- 한국인 설문의 `제주에서 불편했던 점`, `오픈채팅에서 찾는 정보` 상위 값을 보여준다.
- 핵심 해석은 이동 자체뿐 아니라 같은 시간과 목적지의 동행을 찾는 과정에 초점을 둔다.
- 시냅스팟의 파티 탐색·생성·신청 화면을 바로 연결한다.

### 정보 탐색

- 외국인 설문의 `제주에서 불편했던 점`, `오픈채팅에서 찾는 정보` 상위 값을 보여준다.
- 핵심 해석은 정보의 위치, 최신성, 신뢰도 확인 과정에 초점을 둔다.
- 시냅스팟의 근거 있는 AI 질문 화면을 바로 연결한다.

## 화면 구조

1. 응답자 구성 오프닝
2. `전체 응답 / 이동·동행 / 정보 탐색` 세그먼트 컨트롤
3. 선택한 주제의 동적 증거 패널
4. 선택한 주제와 연결된 실제 제품 미리보기
5. 기존 전체 데이터 보기와 PDF 다운로드

세그먼트 컨트롤은 Streamlit 네이티브 `st.segmented_control`을 사용한다. 정적 HTML 내부에 클릭 가능한 척하는 요소를 만들지 않는다.

## 상태와 데이터 흐름

1. 한국인·외국인 설문을 기존 함수로 읽는다.
2. `build_research_view_model()`이 발표용 공통 모델을 만든다.
3. `st.segmented_control`이 `presentation_focus` 값을 결정한다.
4. 별도 함수가 선택값을 `전체`, `이동`, `정보` 패널 모델로 변환한다.
5. 선택 모델을 사용해 근거 패널과 제품 미리보기를 렌더링한다.

허용되는 선택값은 세 가지로 제한한다. 예상하지 못한 값은 `전체 응답`으로 되돌린다. 데이터가 비어 있으면 기존 빈 상태 안내를 표시한다.

## 모션 원칙

- 모션은 선택 결과를 이해시키는 보조 수단으로만 사용한다.
- 막대의 최초 등장 애니메이션은 유지할 수 있지만, 의미 없는 4px 호버 이동과 형제 항목 흐리기는 제거한다.
- 선택 상태는 색상, 굵기, 제목으로 명확히 표시한다.
- 무한 반복, 흔들림, 장식성 파티클은 사용하지 않는다.
- `prefers-reduced-motion` 환경에서는 최종 상태를 즉시 표시한다.

## 접근성과 반응형

- 세그먼트 컨트롤은 키보드와 터치로 사용할 수 있어야 한다.
- 그래프는 항목명, 인원, 비율을 기본 노출한다.
- 색상만으로 선택 상태나 집단을 구분하지 않는다.
- 모바일에서는 컨트롤과 패널을 한 열로 쌓고 제품 이미지 너비를 컨테이너에 맞춘다.

## 검증 계획

- 선택 모델 단위 테스트: 세 선택값이 서로 다른 제목, 데이터, 제품 정보를 반환하는지 확인한다.
- 앱 구성 테스트: `st.segmented_control`과 세션 상태 키가 존재하는지 확인한다.
- 회귀 테스트: 상세 보기, 자유 의견, 새로고침, PDF 다운로드가 유지되는지 확인한다.
- Playwright 동작 테스트:
  - 기본값은 `전체 응답`이다.
  - `이동·동행` 클릭 후 이동 근거와 파티 화면이 나타난다.
  - `정보 탐색` 클릭 후 외국인 근거와 AI 질문 화면이 나타난다.
  - 마우스를 다른 곳으로 옮겨도 선택 결과가 유지된다.
- 390px와 1440px에서 가로 넘침, 텍스트 잘림, 겹침을 확인한다.

## 제외 범위

- 설문 응답 행 단위 교차 분석
- 지도 시각화
- 임의의 점을 특정 문제 응답자로 표현하는 애니메이션
- 시냅스팟 제품 자체의 기능 변경
- 새로운 대형 차트 라이브러리 도입
