# 수요조사 연구 브리프 대시보드 리디자인

## 결정 상태

이 문서는 2026-07-15 시각 목업 검토에서 확정한 방향을 기록한다.

- 정보 구조: `A. 연구 브리프형`
- 시각 언어: `제주 리서치 에디토리얼`
- 차트 언어: 집단별 가로 막대, 응답 인원과 집단 내 비율 병기
- 제품 연결: 설문 분석을 충분히 보여준 뒤 한 번만 크게 연결
- 반응형 기준: 데스크톱과 모바일의 정보 순서를 동일하게 유지

## 제품 정체성

이 사이트는 시냅스팟을 홍보하는 랜딩 페이지가 아니다. 한국인과 외국인 교류학생의 제주 생활 경험을 분석하고, 그 결과가 시냅스팟의 문제 정의로 어떻게 이어졌는지 보여주는 공개 연구 브리프 대시보드다.

첫 화면만 본 방문자는 다음 세 가지를 이해할 수 있어야 한다.

1. 누구를 얼마나 조사했는가
2. 어떤 문제가 반복해서 나타났는가
3. 그 문제가 시냅스팟의 어떤 기능 결정으로 이어졌는가

## 목표

- 발표자가 조사 근거에서 시냅스팟 시연으로 자연스럽게 넘어갈 수 있게 한다.
- 팀원이 현장에서 최신 응답 수를 모바일로 빠르게 확인할 수 있게 한다.
- 공개 방문자가 한국인과 외국인 설문을 구분해 상세 결과를 검토할 수 있게 한다.
- 화면과 PDF가 같은 최신 집계 모델을 사용하게 한다.
- 작은 표본을 시각적으로 과장하지 않으면서도 핵심 결과를 기억하기 쉽게 만든다.

## 제외 범위

- 시냅스팟 서비스 자체의 기능이나 UI 변경
- 설문 문항과 Google Form 구조 변경
- Streamlit 안에 모집, 신청, 채팅 기능을 재구현하는 작업
- 설문이 기능 효과를 검증한 것처럼 보이는 표현
- 응답자 아이콘 39개, 세계지도, 장식용 도넛처럼 표본을 크게 보이게 하는 인포그래픽
- 발표 주제를 선택하면 화면이 바뀌는 시나리오 버튼
- 의미 없는 호버 이동, 반복 재생 애니메이션, 장식용 그래디언트

## 전체 정보 구조

페이지는 다음 순서를 고정한다.

1. 조사 헤더와 실시간 응답 현황
2. 먼저 볼 결론
3. 응답이 가리킨 두 가지 문제
4. 한국인과 외국인의 경험 비교
5. 조사 결과에서 시냅스팟 기능으로 연결
6. 상세 조사 결과
7. 익명 자유 의견과 공유 자료

모바일에서는 각 영역을 한 열로 쌓고, 데스크톱에서는 비교가 필요한 영역만 두 열로 배치한다. 화면 폭이 바뀌어도 정보 순서는 바꾸지 않는다.

## 화면 설계

### 1. 조사 헤더와 응답 현황

- 상단 표기: `JEJU EXCHANGE STUDENT RESEARCH`
- 제목: `교류학생의 이동과 정보 탐색은 어디서 막혔을까?`
- 설명: 실제 경험 조사와 문제 정의의 관계를 한 문장으로 설명한다.
- 지표: 전체 응답, 한국인 응답, 외국인 응답, 데이터 상태
- 보조 정보: 마지막 갱신 시각과 데이터 새로고침 버튼

전체 응답 수는 가장 크게 표시하지만 영웅 섹션 크기로 키우지 않는다. 모바일 390px 기준으로 제목, 응답 현황, 새로고침 버튼이 첫 화면 안에 들어와야 한다.

### 2. 먼저 볼 결론

한 문장만 사용한다.

`이동·동행 모집과 신뢰할 수 있는 생활정보 탐색이 함께 필요했습니다.`

이 문장은 분석의 방향을 요약하는 해석이며, 특정 기능의 효과를 입증했다는 표현은 사용하지 않는다. 옅은 산호색 배경과 왼쪽 강조선으로 본문과 구분한다.

### 3. 응답이 가리킨 두 가지 문제

두 문제를 동일한 시각적 비중으로 배치한다.

#### 문제 1: 이동과 동행 모집

- 한국인 설문의 버스 노선, 택시비, 택시팟 탐색
- 외국인 설문의 교통, 택시비, 동행 관련 응답
- 관련 항목의 응답 인원과 집단 내 비율

#### 문제 2: 공지와 생활정보 탐색

- 외국인 설문의 공지 탐색
- 한국인과 외국인의 정보 부족 및 원하는 글 찾기 어려움
- 오픈채팅의 정보 탐색 불편

데이터 항목은 현재 집계 순위에서 가져오며 특정 수치를 HTML이나 CSS에 고정하지 않는다.

### 4. 한국인과 외국인의 경험 비교

한국인과 외국인을 경쟁시키거나 단순 합산하지 않는다. 각 집단의 표본 수를 함께 표시하고, 집단별 상위 신호와 공통 문제를 나란히 보여준다.

- 한국인: `n=현재 한국인 응답 수`
- 외국인: `n=현재 외국인 응답 수`
- 비율 분모: 각 집단의 응답 수
- 전체 비교: 응답 인원과 집단 내 비율을 모두 표시

표본 수가 다르므로 막대 길이만으로 두 집단의 절대 규모를 비교하게 만들지 않는다.

### 5. 조사 결과에서 시냅스팟 기능으로 연결

이 영역은 모든 핵심 분석 뒤에 한 번만 표시한다. 앞선 그래프마다 제품 기능을 붙이지 않는다.

- 표기: `FROM RESEARCH TO PRODUCT`
- 연결 1: 이동·동행 수요 → `이동·동행 파티`
- 연결 2: 공지·생활정보 탐색 → `근거 있는 정보 탐색`

목업이 아니라 시냅스팟의 실제 모바일 화면 두 장을 사용한다. 자산은 시냅스팟 저장소의 현재 구현을 확인한 뒤 이 프로젝트의 `assets/`에 발표 시점 스냅샷으로 복사한다. 화면이 없거나 로드에 실패하면 기능명과 설명만 남아도 레이아웃이 유지되어야 한다.

이 영역의 배경은 짙은 남색을 사용해 분석과 제품 연결의 전환점을 만든다. 별도의 발표 주제 선택 버튼이나 가짜 인터랙션은 넣지 않는다.

### 6. 상세 조사 결과

다음 네 보기를 탭 형태로 유지한다.

- 전체 요약
- 한국인 설문
- 외국인 설문
- 비교 요약

탭은 제품 연결 영역 아래에 둔다. 모바일에서는 두 줄로 자연스럽게 배치하거나 가로 스크롤 가능한 탭으로 만들되, 글자가 잘리지 않아야 한다.

기존의 활동 수요, 오픈채팅 탐색, 오픈채팅 불편, 서비스 사용 의향을 모두 유지한다. 상세 결과에서는 새로운 해석을 반복하기보다 수치와 항목을 검토할 수 있게 한다.

### 7. 익명 자유 의견과 공유 자료

- 한국인과 외국인 의견을 모두 표시한다.
- 한 번에 최대 5개를 표시하고 페이지를 이동할 수 있게 한다.
- 빈 문자열과 개인정보로 보이는 값은 노출하지 않는다.
- PDF는 다운로드 시점의 현재 집계로 생성한다.
- PDF에 전체, 한국인, 외국인 응답 수와 데이터 기준 시각을 명확히 표시한다.

## 시각 시스템

### 색상

- 본문 남색: `#13233D`
- 바다색: `#087F72`
- 산호색: `#EC6A5F`
- 배경: `#FBFCFD`
- 구분선: `#D8E2E9`
- 보조 텍스트: `#66768A`
- 제품 연결 배경: `#132E50`

남색은 제목과 제품 연결에, 바다색은 외국인 및 실시간 상태와 주요 막대에, 산호색은 한국인 및 결론 강조에 사용한다. 색만으로 집단이나 상태를 구분하지 않고 레이블을 함께 표시한다.

### 타이포그래피

- 운영체제 기본 산세리프 또는 프로젝트에 이미 포함된 한글 산세리프를 사용한다.
- 데스크톱 제목은 약 40px, 모바일 제목은 약 32px 범위로 고정한다.
- 뷰포트 너비에 따라 글자 크기가 연속적으로 변하는 `vw` 기반 크기는 사용하지 않는다.
- 카드와 패널 내부 제목은 본문보다 한 단계만 크게 사용한다.
- 자간은 0으로 유지한다.

### 형태와 간격

- 카드와 패널의 모서리 반경은 8px 이하로 제한한다.
- 전체 페이지 섹션을 떠 있는 카드로 만들지 않는다.
- 카드는 반복 지표, 의견, 실제 제품 화면처럼 개별 항목에만 사용한다.
- 모바일 좌우 여백은 최소 16px, 데스크톱 본문 최대 폭은 약 960px로 둔다.
- 같은 계층의 섹션은 동일한 상하 간격을 사용한다.

## 차트 언어

### 기본 규칙

- 순위형 응답은 가로 막대를 사용한다.
- 모든 막대에 항목명, 응답 인원, 집단 내 비율을 직접 표시한다.
- 파이와 도넛은 비율 합계가 의미 있는 단일 응답 문항에만 제한적으로 사용한다.
- 서비스 사용 의향은 긍정, 중립, 부정의 분할 막대로 표시한다.
- 긴 항목명은 줄바꿈하며 말줄임표로 핵심 내용을 숨기지 않는다.
- 자유 의견은 차트로 변환하지 않는다.

### 백분율 규칙

- 한국인 차트 분모는 한국인 응답 수다.
- 외국인 차트 분모는 외국인 응답 수다.
- 복수 응답 문항은 비율 합계가 100%를 넘을 수 있음을 표시한다.
- 전체 합산 비율이 필요하면 전체 응답 수를 분모로 사용했다는 사실을 제목이나 보조 문구에 적는다.
- 응답 수가 0이면 비율은 `0%`로 표시하고 순위 해석을 만들지 않는다.

## 상호작용과 애니메이션

- 첫 진입 기본 보기는 `전체 요약`이다.
- 데이터 새로고침은 Streamlit 캐시를 비우고 두 설문을 다시 읽은 뒤 전체 화면을 재계산한다.
- 차트 막대는 최초 진입 시 짧게 한 번 채워질 수 있다.
- `prefers-reduced-motion` 사용자는 애니메이션을 끈다.
- 호버 시 위치가 움직이거나 크기가 변하는 효과는 사용하지 않는다.
- 다운로드와 새로고침 외에 명령형 버튼을 추가하지 않는다.

## 데이터 흐름과 불변 조건

데이터 흐름은 다음 하나로 통일한다.

`Google Sheets 또는 CSV → 설문별 정규화 → 공통 집계 객체 → 웹 화면과 PDF`

불변 조건은 다음과 같다.

1. 화면과 PDF는 같은 실행에서 생성한 집계 객체를 받는다.
2. 응답 수와 비율을 화면 문구에 고정하지 않는다.
3. 한국인과 외국인 모두 `Google Sheets`일 때만 `실시간 집계`로 표시한다.
4. 한쪽이라도 CSV 대체 데이터면 정확한 소스 상태를 표시한다.
5. 새로고침 후 헤더, 차트, 해석 문장, PDF 입력이 모두 같은 데이터로 바뀐다.
6. 빈 데이터와 누락 문항은 0 또는 빈 목록으로 안전하게 처리한다.

## 코드 경계

- `app.py`: Streamlit 페이지 설정, 데이터 로드, 위젯 상태, 화면 조립
- `src/survey_dashboard.py`: Google Sheets 및 CSV 로드, 설문 파싱과 집계
- 새 리서치 UI 모듈: 헤더, 결론, 문제, 비교, 제품 연결용 순수 HTML 생성 함수
- 기존 PDF 생성 로직: 현재 동작을 유지하되 웹과 같은 집계 객체를 입력으로 받는지 검증

`app.py` 전체를 한 번에 재작성하지 않는다. 먼저 순수 HTML 생성 함수를 작은 단위로 추가하고, 기존 상세 결과와 다운로드를 유지한 채 상단 연구 브리프 영역을 교체한다.

## 오류와 대체 상태

- Sheets 연결 실패: 저장 CSV를 표시하고 노란 안내 영역에 소스와 마지막 저장 시각을 표시한다.
- 일부 설문만 연결 실패: 한국인과 외국인의 소스 상태를 따로 표시한다.
- 데이터 없음: 응답 수 0명과 수집 대기 상태를 표시하고 결론 문장을 숨긴다.
- 제품 화면 자산 없음: 기능명과 설명만 표시하고 깨진 이미지 아이콘을 노출하지 않는다.
- PDF 생성 실패: 웹 대시보드는 유지하고 다운로드 영역에 짧은 오류와 재시도 방법을 표시한다.
- 긴 문항과 의견: 고정 높이로 자르지 않고 줄바꿈한다.

## 접근성과 모바일 기준

- 본문과 배경의 색 대비를 WCAG AA 수준으로 유지한다.
- 차트 값은 색과 막대만이 아니라 텍스트로도 읽을 수 있어야 한다.
- 실제 제품 이미지에는 화면 기능을 설명하는 대체 텍스트를 넣는다.
- 키보드로 탭, 새로고침, 의견 페이지, 다운로드를 사용할 수 있어야 한다.
- 390px 화면에서 가로 스크롤이 없어야 한다.
- 첫 모바일 화면에 제목, 전체·한국인·외국인 응답 수, 데이터 상태, 새로고침이 보여야 한다.
- Streamlit 소유자에게만 보이는 `Manage app` UI는 제품 레이아웃 검증 대상에서 제외한다.

## 검증 전략

### 자동 검증

- 헤더가 한국인과 외국인 응답 수를 합산하는지 확인한다.
- 두 소스가 Sheets일 때만 실시간 상태가 표시되는지 확인한다.
- 집단별 비율이 각 집단 응답 수를 분모로 사용하는지 확인한다.
- 0명, 빈 목록, 일부 소스 실패 상태를 단위 테스트한다.
- 화면과 PDF에 전달되는 응답 수가 일치하는지 확인한다.
- 기존 익명 의견 5개 페이지네이션과 상세 보기 선택을 회귀 테스트한다.

### 브라우저 검증

- 데스크톱 1440x900과 모바일 390x844에서 스크린샷을 확인한다.
- 가로 넘침, 텍스트 잘림, 버튼과 탭 겹침이 없는지 확인한다.
- 첫 화면에서 조사 대상, 응답 수, 데이터 상태를 읽을 수 있는지 확인한다.
- 실제 제품 화면이 로드되고 분석 영역을 가리지 않는지 확인한다.
- 새로고침 후 최신 Google Sheets 응답 수가 반영되는지 확인한다.

## 완료 조건

- 첫 화면만 보고 사이트가 수요조사 분석 대시보드임을 알 수 있다.
- 전체, 한국인, 외국인 응답 수가 최신 Google Sheets 집계와 일치한다.
- 두 핵심 문제가 실제 설문 수치와 함께 설명된다.
- 시냅스팟 연결은 분석 뒤 한 영역에서만 나타난다.
- 제품 연결 영역은 실제 모바일 화면 두 장을 사용한다.
- 상세 보기, 익명 의견, 새로고침, PDF 기능이 유지된다.
- 화면과 PDF의 응답 수 및 데이터 기준이 일치한다.
- 모바일과 데스크톱 검증을 모두 통과한다.
