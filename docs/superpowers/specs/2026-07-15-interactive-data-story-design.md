# 발표형 인터랙티브 데이터 스토리 설계

## 목적

현재 수요조사 공개 페이지를 정적인 보고서형 대시보드에서 발표자가 직접 스크롤하며 설명할 수 있는 데이터 스토리로 발전시킨다. 방문자는 첫 화면부터 전체 응답 규모와 두 핵심 문제를 이해하고, 움직이는 시각화를 따라가며 조사 결과가 시냅스팟의 실제 기능으로 연결되는 과정을 확인한다.

기존 Google Sheets 실시간 집계, 한국인/외국인 상세 통계, 익명 자유 의견, PDF 다운로드는 유지한다. 애니메이션은 데이터를 흥미롭게 전달하기 위한 수단으로만 사용하고, 설문이 기능 효과를 검증했다는 오해를 만들지 않는다.

## 성공 기준

- 발표자가 별도의 슬라이드 없이 상단부터 스크롤하며 `응답자 구성 → 두 문제 → 조사 근거 → 시냅스팟 기능` 순서로 설명할 수 있다.
- 첫 화면에서 전체 39명처럼 현재 집계된 총응답 수와 한국인/외국인 구성을 즉시 이해할 수 있다.
- 데이터 시각화는 실제 집계값으로 생성되며 특정 인원이나 비율을 코드에 고정하지 않는다.
- 모바일과 데스크톱에서 텍스트, 점, 그래프, 제품 미리보기가 잘리거나 겹치지 않는다.
- 애니메이션을 지원하지 않거나 동작 감소를 선호하는 환경에서도 모든 정보가 정적으로 읽힌다.
- 기존 상세 통계, 자유 의견 페이지네이션, 새로고침, PDF 다운로드가 그대로 동작한다.

## 발표 흐름

### 1. 오프닝: 응답자 구성

- 제목은 `39명의 응답이 두 가지 문제를 가리켰습니다` 형태로 현재 총응답 수를 반영한다.
- 전체 응답 수만큼 원형 점을 표시한다.
- 점은 처음에 하나의 무리로 등장한 뒤 한국인과 외국인 두 그룹으로 나뉜다.
- `한국인 N명`, `외국인 N명`, 마지막 집계 시각을 함께 표시한다.
- 표본 수가 많아져 점을 개별 표시하기 어려워지면 최대 60개까지만 표시하고, 점 하나가 여러 응답을 나타낸다는 범례를 제공한다.

### 2. 발견: 두 가지 문제

- 점들이 코랄 계열의 `이동·동행 모집`과 청록 계열의 `공지·생활정보 탐색` 두 영역으로 시각적으로 이어진다.
- 각 문제는 한 문장 설명, 대표 수치 두 개, 관련 설문 집단을 포함한다.
- 대표 수치는 현재 한국인/외국인 설문의 상위 응답에서 계산한다.
- 두 문제를 경쟁 순위로 표현하지 않고, 서로 다른 학생 경험에서 반복된 문제로 설명한다.

### 3. 문제 1: 이동·동행 모집

- 한국인 설문의 `불편했던 점`과 `오픈채팅에서 찾는 정보`를 큰 수평 막대그래프로 보여준다.
- 막대는 섹션이 처음 보일 때 한 번 차오른다.
- 막대에 마우스를 올리거나 키보드로 초점을 이동하면 정확한 응답 수와 비율을 표시하고 나머지 막대는 약하게 흐려진다.
- 모바일에서는 탭 없이 상위 5개 항목을 한 열로 표시하며, 터치하면 같은 상세값을 보여준다.
- 핵심 해석은 `비용과 이동 자체뿐 아니라 같은 시간과 목적지의 사람을 찾는 과정이 반복된다`로 제한한다.

### 4. 문제 2: 공지·생활정보 탐색

- 외국인 설문의 `불편했던 점`과 `오픈채팅에서 찾는 정보`를 같은 시각 문법으로 보여준다.
- 이동·동행 섹션과 색은 구분하되 그래프 구조와 상호작용은 동일하게 유지한다.
- 핵심 해석은 `정보를 찾는 것뿐 아니라 최신성과 신뢰도를 확인하는 과정이 필요하다`로 제한한다.

### 5. 전환: 조사에서 제품으로

- 두 문제의 색상 흐름이 각각 시냅스팟 기능으로 연결되는 짧은 브리지 섹션을 둔다.
- `이동·동행 모집 → 파티 탐색·생성·신청·채팅`
- `공지·생활정보 탐색 → AI 질문·근거 검색·출처·신뢰도`
- 연결은 제품 기획 해석이며 설문이 기능 효과를 입증한 것은 아니라는 문구를 유지한다.

### 6. 실제 제품 미리보기

- 시냅스팟의 파티 화면과 AI 질문 화면을 실제 모바일 화면 이미지로 보여준다.
- 데스크톱에서는 두 화면을 좌우로 배치하고, 모바일에서는 문제 설명 다음에 해당 화면이 바로 이어지도록 세로로 배치한다.
- 이미지에 가벼운 등장 효과를 적용하되 자동 확대, 무한 반복, 흔들림 효과는 사용하지 않는다.

### 7. 상세 조사와 공유 자료

- 기존 `전체 요약`, `한국인 설문`, `외국인 설문`, `비교 요약` 선택 기능을 유지한다.
- 발표 흐름과 시각적으로 분리하기 위해 `전체 데이터 살펴보기` 구간 아래에 배치한다.
- 익명 자유 의견, 새로고침, 현재 데이터 PDF 다운로드를 유지한다.

## 시각 디자인

### 기본 인상

- 대학생 대상의 밝고 현대적인 에디토리얼 인포그래픽을 지향한다.
- 흰색과 아주 옅은 회청색을 배경으로 사용하고, 짙은 남색을 본문과 제목에 사용한다.
- 코랄은 이동·동행, 청록은 정보 탐색, 파랑은 중립 데이터와 상호작용 상태에 사용한다.
- 한 화면에 여러 장식 카드를 쌓지 않고, 섹션 하나가 메시지 하나를 담당한다.

### 타이포그래피

- 첫 화면의 수치와 핵심 문장은 크게 표시하되 모바일에서 문장이 두세 줄 안에 들어오도록 제한한다.
- 그래프 레이블과 값은 본문보다 작지만 최소 14px 이상을 유지한다.
- 영문 키커는 보조 요소로만 사용하며 한글 제목보다 강하게 보이지 않도록 한다.

### 모션

- 최초 등장: 점이 순차적으로 나타난 뒤 두 집단으로 분리된다.
- 섹션 진입: 막대가 0에서 실제 비율까지 한 번 차오른다.
- 상호작용: 선택한 항목은 색과 굵기로 강조하고 나머지는 투명도를 낮춘다.
- 제품 전환: 두 문제의 색상 선이 각 기능 미리보기 방향으로 이어진다.
- 모션 시간은 250~700ms 범위로 제한하고 무한 반복 애니메이션은 사용하지 않는다.
- `prefers-reduced-motion: reduce` 환경에서는 이동 효과를 제거하고 최종 상태를 즉시 표시한다.

## 기술 설계

### 구성 요소

1. `build_research_view_model()`
   - 기존 한국인/외국인 집계 객체를 발표형 시각화에 필요한 데이터 모델로 확장한다.
   - 총응답, 집단별 인원, 대표 문제, 상위 그래프 항목, 서비스 긍정 의향을 제공한다.

2. 응답자 점 시각화
   - SVG를 직접 그리지 않고 HTML 요소와 CSS Grid를 사용한다.
   - 점 개수, 색상, 접근성 레이블은 서버에서 생성한다.
   - CSS 애니메이션만 사용해 Streamlit 재실행과 브라우저 보안 제약의 영향을 줄인다.

3. 인터랙티브 막대그래프
   - 기존 HTML 막대그래프를 확장해 CSS hover, focus, tooltip을 제공한다.
   - 별도 대형 차트 라이브러리 도입 없이 실제 값과 비율을 항상 텍스트로 병기한다.
   - JavaScript가 없어도 핵심 정보가 모두 노출되는 구조를 우선한다.

4. 스토리 섹션
   - 오프닝, 두 문제, 제품 연결을 `src/research_page.py`의 독립 렌더링 함수로 분리한다.
   - 상세 대시보드 렌더링은 `app.py`에 유지하되 발표 흐름 아래에 배치한다.

5. 제품 미리보기
   - 기존 시냅스팟 이미지 자산과 누락 시 대체 레이아웃을 유지한다.
   - 새 애니메이션은 CSS 클래스만 추가하며 이미지 자체를 수정하지 않는다.

### 데이터 흐름

1. Google Sheets 또는 CSV 대체 데이터에서 한국인/외국인 설문을 읽는다.
2. 기존 파서가 설문별 집계를 생성한다.
3. `build_research_view_model()`이 같은 집계를 발표형 스토리 모델로 변환한다.
4. 오프닝, 문제 시각화, 상세 통계, PDF가 같은 집계 객체를 사용한다.
5. 새로고침 시 Streamlit 캐시를 비우고 모든 시각화 값을 다시 계산한다.

## 반응형 동작

### 데스크톱

- 최대 콘텐츠 너비는 약 1180px로 제한한다.
- 오프닝 점 시각화와 집단 수치를 한 화면에 함께 보여준다.
- 문제 섹션은 설명과 그래프를 2열로 구성할 수 있다.
- 제품 미리보기는 두 모바일 화면을 나란히 배치한다.

### 모바일

- 모든 섹션을 한 열로 쌓는다.
- 점 시각화는 컨테이너 너비에 맞춰 열 수를 자동 조정한다.
- 그래프 레이블, 막대, 값을 세로 구조로 바꿔 긴 문구가 잘리지 않게 한다.
- hover 전용 정보는 사용하지 않고 값과 비율을 기본 노출한다.
- 애니메이션 이동 거리를 줄여 화면 흔들림과 레이아웃 이동을 방지한다.

## 접근성과 오류 처리

- 모든 그래프 값은 색만으로 구분하지 않고 레이블, 인원, 비율을 함께 표시한다.
- 점 시각화는 전체 응답과 집단별 인원을 설명하는 접근성 텍스트를 포함한다.
- 키보드 초점이 가능한 그래프 항목에는 명확한 포커스 테두리를 표시한다.
- 데이터가 비어 있으면 점과 그래프 대신 `아직 집계된 응답이 없습니다`를 표시한다.
- Google Sheets 연결 실패 시 기존 CSV 대체 안내를 유지한다.
- 긴 항목명, 0명 응답, 표본 증가가 레이아웃을 깨지 않도록 처리한다.

## 성능과 구현 경계

- 대형 시각화 라이브러리와 실시간 캔버스 렌더러는 추가하지 않는다.
- HTML/CSS 기반 표현을 우선해 Streamlit Community Cloud에서 추가 서버 설정 없이 동작하게 한다.
- 전체 응답 점은 최대 60개만 렌더링해 표본 증가 시에도 DOM 크기를 제한한다.
- synapspot 저장소와 실제 제품 기능은 수정하지 않는다.
- 기존 PDF 양식은 이번 작업에서 변경하지 않는다.

## 검증 계획

- Python 문법 검사와 전체 단위 테스트를 실행한다.
- 동적 응답 수를 사용하고 하드코딩된 `39명`이 없는지 테스트한다.
- 한국인/외국인 데이터가 0명인 경우와 CSV 대체 경로를 테스트한다.
- 390px 모바일과 1440px 데스크톱에서 Playwright 스크린샷을 확인한다.
- 가로 스크롤, 텍스트 잘림, 요소 겹침, 빈 그래프가 없는지 확인한다.
- 모션 감소 환경에서 최종 상태가 즉시 보이는지 확인한다.
- 공개 미리보기 배포 후 Google Sheets 데이터와 화면 집계가 일치하는지 확인한다.
- 상세 보기, 자유 의견 페이지네이션, 새로고침, PDF 다운로드 회귀 테스트를 실행한다.

## 제외 범위

- 지도 기반 시각화
- 여러 그래프가 서로 필터를 바꾸는 분석 도구
- 사용자가 설문 원본 행을 탐색하는 기능
- 자동 재생되는 발표 슬라이드
- 시냅스팟 제품 화면이나 기능 자체의 리디자인
- 장식 목적의 3D, 파티클 배경, 무한 반복 모션
