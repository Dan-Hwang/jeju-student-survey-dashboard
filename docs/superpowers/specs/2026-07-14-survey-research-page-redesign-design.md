# 수요조사 공개 리서치 페이지 리디자인

## 목적

기존 Streamlit 수요조사 대시보드를 시냅스팟의 문제 정의와 자연스럽게 이어지는 공개 리서치 페이지로 재구성한다. Google Sheets 실시간 집계, 한국인/외국인 비교, 익명 자유 의견, 최신 데이터 PDF 생성 기능은 유지한다.

이 페이지는 시냅스팟 서비스 자체를 대신하지 않는다. 설문으로 확인한 문제와 실제 서비스 기능의 연결을 보여주고, 발표자가 조사 근거에서 서비스 시연으로 매끄럽게 넘어갈 수 있도록 돕는다.

## 범위

### 포함

- 첫 화면을 문제 제시와 핵심 조사 결과 중심으로 재구성
- 한국인/외국인 설문에서 계산한 핵심 지표 자동 표시
- 조사 결과와 시냅스팟 기능의 연결 설명
- 시냅스팟의 AI 질문 및 파티 기능 미리보기
- 모바일 단일 열, 데스크톱 2열 반응형 구성
- 기존 상세 통계, 익명 의견 페이지네이션, 데이터 새로고침 유지
- 기존 실시간 PDF 생성 유지
- Google Sheets 실패 시 CSV 대체 결과와 오류 안내 유지

### 제외

- synapspot 저장소의 기능 또는 디자인 변경
- 설문 문항이나 Google Form 구조 변경
- 사용자 인증, 서비스 신청, 채팅 기능을 Streamlit에 재구현
- 설문 결과로 뒷받침되지 않는 기능 효과 주장

## 사용자와 발표 흐름

1. 방문자는 제주에 처음 온 학생이 정보와 동행을 따로 찾아야 하는 문제를 확인한다.
2. 전체 응답 수와 한국인/외국인 핵심 수치를 한눈에 본다.
3. 조사에서 반복된 문제를 `이동·동행 모집`과 `공지·생활정보 탐색` 두 축으로 이해한다.
4. 각 문제와 시냅스팟의 실제 기능이 어떻게 연결되는지 확인한다.
5. AI 질문 화면과 파티 화면 미리보기를 통해 해결 방식을 구체적으로 본다.
6. 필요하면 한국인·외국인·비교 상세 통계와 익명 의견을 탐색한다.
7. 최신 집계 결과가 포함된 PDF를 내려받는다.

## 화면 구조

### 1. 리서치 헤더

- 서비스명보다 조사 목적을 먼저 전달한다.
- 제목: `교류학생의 제주 생활, 무엇이 가장 불편했을까?`
- 설명: 한국인·외국인 교류학생의 이동, 동행 모집, 생활정보 탐색 경험을 조사했다는 내용을 짧게 표시한다.
- 전체 응답 수, 한국인 응답 수, 외국인 응답 수, 마지막 갱신 시각을 간결하게 표시한다.
- 데이터 새로고침은 보조 아이콘 버튼 또는 작은 명령 버튼으로 둔다.

### 2. 핵심 조사 결과

- 전체 응답 수
- 한국인 설문의 핵심 불편
- 외국인 설문의 핵심 불편
- 두 집단의 서비스 긍정 의향

수치는 Google Sheets에서 읽은 현재 집계를 사용한다. 특정 응답 수나 비율을 코드에 고정하지 않는다.

### 3. 발견한 두 가지 문제

#### 이동·동행 모집

택시팟, 같이 이동할 사람, 활동 동행 수요를 관련 지표와 함께 보여준다. 데이터가 바뀌더라도 항목명과 비율은 현재 설문 집계를 기준으로 표시한다.

#### 공지·생활정보 탐색

공지, 생활정보, 원하는 글 찾기, 채팅 흐름 문제를 관련 지표와 함께 보여준다. 한국인과 외국인의 차이는 경쟁 구도가 아니라 필요한 지원 방식의 차이로 설명한다.

### 4. 조사 결과에서 기능으로

두 개의 연결 블록을 사용한다.

- `이동·동행 수요` → 파티 목록, 조건별 탐색, 모임 생성, 신청·승인·채팅
- `공지·생활정보 탐색` → 자연어 질문, 커뮤니티 근거 검색, 출처와 신뢰도 표시

연결 문구는 제품 기획 해석임을 분명히 하고, 설문이 기능 효과를 입증한 것처럼 표현하지 않는다.

### 5. 시냅스팟 미리보기

- 실제 `QuestionScreen`과 `MeetingsScreen`을 대표하는 모바일 화면 2개를 사용한다.
- 화면이 준비되지 않았거나 로드되지 않으면 기능명과 설명만으로도 레이아웃이 무너지지 않아야 한다.
- 미리보기는 제품 전체 설명이 아니라 조사 결과와 직접 연결되는 기능만 보여준다.
- 실제 서비스 주소가 확정된 경우에만 명확한 이동 버튼을 노출한다.

### 6. 상세 조사 결과

보기 선택은 다음 네 가지를 유지한다.

- 전체 요약
- 한국인 설문
- 외국인 설문
- 비교 요약

상세 통계는 첫 화면보다 시각적 우선순위를 낮추되, 현재 막대그래프, 사용 의향 도표, 익명 의견 페이지네이션을 유지한다.

### 7. 공유 자료

- PDF는 다운로드 시점의 한국인/외국인 현재 집계로 생성한다.
- 데이터 소스와 마지막 갱신 시각을 표시한다.
- 원본 응답 행과 개인 식별 정보는 포함하지 않는다.

## 시각 원칙

- 대학생 대상의 밝고 차분한 제품 리서치 스타일을 사용한다.
- 흰색과 옅은 회색을 기본으로, 시냅스팟 화면과 충돌하지 않는 청록·파랑을 제한적으로 사용한다.
- 큰 장식보다 실제 수치, 짧은 문장, 명확한 계층을 우선한다.
- 카드 반경은 8px 이하를 기본으로 하고, 반복 지표 카드만 제한적으로 사용한다.
- 모바일에서 텍스트와 그래프가 화면 밖으로 넘치지 않아야 한다.
- 모바일에서는 모든 핵심 영역을 한 열로 쌓고, 데스크톱에서만 비교 영역을 2열로 배치한다.
- 애니메이션은 최초 등장과 상태 변화에만 짧게 사용하며 발표 집중을 방해하지 않는다.
- Streamlit 기본 위젯의 목적이 불명확하지 않도록 레이블과 상태를 정리한다.

## 데이터 흐름

1. `get_public_survey()`와 `get_foreign_survey()`가 Google Sheets 또는 CSV 대체 데이터를 읽는다.
2. 기존 파서가 응답 수, 불편, 활동, 오픈채팅 탐색, 오픈채팅 불편, 사용 의향, 자유 의견을 집계한다.
3. 리서치 헤더와 핵심 결과는 같은 집계 객체에서 값을 가져온다.
4. 상세 통계와 PDF도 동일한 집계 객체를 사용한다.
5. 사용자가 데이터 새로고침을 누르면 Streamlit 캐시를 비우고 전체 화면을 다시 계산한다.

화면과 PDF가 서로 다른 파일이나 오래된 정적 결과를 참조하지 않도록 한다.

## 오류 처리

- Google Sheets 연결 실패 시 저장된 CSV 결과를 표시하고 데이터 기준을 눈에 띄게 안내한다.
- 한국인 또는 외국인 데이터가 비어 있으면 해당 수치를 0명으로 표시하고 해석 문장을 단정적으로 만들지 않는다.
- 시냅스팟 미리보기 자산이 없더라도 상세 조사와 PDF 기능은 정상 동작해야 한다.
- 긴 항목명과 자유 의견은 모바일에서 줄바꿈하며, 고정 높이 iframe 안에서 잘리지 않도록 한다.

## 구현 경계

현재 `app.py`에 집중된 화면 스타일과 PDF 코드는 이번 리디자인 범위에서 필요한 만큼만 분리한다.

- 리서치 페이지 UI와 차트 HTML은 별도 모듈로 옮길 수 있다.
- Google Sheets 파싱 로직은 `src/survey_dashboard.py`에 유지한다.
- PDF 생성은 동작 변경이 없다면 기존 로직을 유지한다.
- synapspot 저장소는 읽기 전용 참고 대상으로 사용한다.

## 검증 기준

- Python 문법 검사와 기존 단위 테스트가 통과한다.
- 한국인/외국인 데이터가 Google Sheets 및 CSV 대체 경로에서 모두 렌더링된다.
- PDF의 응답 수와 화면의 응답 수가 일치한다.
- 390px 안팎 모바일 화면에서 가로 스크롤과 텍스트 잘림이 없다.
- 데스크톱에서 핵심 결과와 기능 연결이 2열로 자연스럽게 보인다.
- 첫 화면만 보고도 `조사 대상`, `핵심 문제`, `시냅스팟과의 연결`을 이해할 수 있다.
- 익명 자유 의견은 한 페이지에 최대 5개만 표시된다.

## 완료 조건

- 발표자는 수요조사 결과에서 시냅스팟의 AI 질문 또는 파티 데모로 별도 설명 없이 이어갈 수 있다.
- 공개 방문자는 상세 통계와 PDF를 통해 조사 근거를 다시 확인할 수 있다.
- 새 응답이 추가되면 핵심 지표, 상세 통계, PDF가 같은 데이터 기준으로 갱신된다.
